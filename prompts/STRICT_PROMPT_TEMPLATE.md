# Strict Prompt Template

所有游戏 prompt 使用下列章节顺序。目标是降低模型自由补全经典模板的空间，并让不同模型面对同一份可验证规格。

## Required Sections

1. `任务身份`：使用独立任务名，明确禁止套用经典游戏默认规则。
2. `输出契约`：限定 Python、pygame、单文件、无外部资源、只输出代码。
3. `坐标与时间`：固定窗口、FPS、左上角原点、矩形边界语义、内部浮点和绘制取整规则。
4. `颜色表`：为每个可见对象固定 RGB，不接受“明显区分”等模糊描述。
5. `布局与实体`：列出所有区域、尺寸、起点、固定序列或固定地图。
6. `输入`：逐键说明按下、持续按住和禁止行为。
7. `更新顺序`：逐帧或逐回合明确先后顺序，避免碰撞和计分歧义。
8. `独特规则`：至少一个偏离公开经典模板的强制机制。
9. `计分与终止`：列出所有分数变化、胜负条件、终局冻结和重开行为。
10. `绘制顺序与 HUD`：固定层级、文字、位置和字号。
11. `禁止项`：禁止额外模式、素材、模糊替代实现。
12. `提交前检查`：要求模型自检，但最终只能输出代码。

## Precision Rules

- 坐标使用 `(x, y)`，尺寸使用 `(width, height)`，单位固定为像素。
- 网格任务同时写明网格坐标与像素换算公式。
- 所有随机行为写出种子；能使用固定序列时优先固定序列。
- 禁止使用“附近”“大约”“合理范围”“建议”“适当”“少量”“低频”等词。
- 碰撞必须说明使用 `pygame.Rect`、中心点还是圆形距离。
- 终局后必须停止状态演化，只保留事件处理、绘制、`R` 和 `ESC`。
