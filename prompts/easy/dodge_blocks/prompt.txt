任务身份
请使用 Python 3 和 pygame 实现单文件游戏 `Lane Cipher`。它是固定车道躲避与收集游戏，不是自由移动的标准 Dodge Blocks 模板。

输出契约
1. 只输出完整 Python 代码，不要输出解释、Markdown 或代码块标记。
2. 禁止任何外部资源。只使用 pygame 图元和 `pygame.font.SysFont(None, size)`。
3. 所有常量和生成序列写在顶部。调用 `random.seed(42)`，但实体生成必须使用固定序列。

坐标与时间
1. 窗口固定 `800x600`，标题固定 `Lane Cipher`，帧率固定 `60 FPS`。
2. 游戏区域固定为 `pygame.Rect(80,80,640,480)`，共 `8` 条车道，每条宽 `80`。车道 `lane` 的左边为 `80 + lane*80`。
3. 所有实体使用整数 `pygame.Rect` 碰撞。

颜色表
`BACKGROUND=(12,18,30)`，`FIELD=(24,34,52)`，`LANE_LINE=(54,72,98)`，`PLAYER=(86,224,154)`，`HAZARD=(255,92,112)`，`HAZARD_EDGE=(150,38,62)`，`CHIP=(255,196,72)`，`CHIP_EDGE=(255,238,160)`，`TEXT=(238,244,255)`。

布局与实体
1. 玩家矩形固定大小 `54x34`，初始车道为 `3`，初始矩形为 `pygame.Rect(333,512,54,34)`。
2. 危险块固定大小 `44x44`；芯片固定大小 `24x24`。生成时实体在目标车道内水平居中，顶部固定为 `y=80-实体高度`。
3. 每隔 `36` 帧生成一个实体。生成序列固定循环使用：
`[("hazard",1,5),("hazard",5,6),("chip",3,5),("hazard",7,7),("hazard",2,5),("hazard",4,8),("chip",6,6),("hazard",0,7),("hazard",3,6),("hazard",6,8),("chip",1,5),("hazard",5,7)]`。
4. 三元组依次表示 `(kind,lane,speed_pixels_per_frame)`。

输入
1. 仅在 `KEYDOWN` 时响应左方向键或 `A`：车道编号减 `1`；右方向键或 `D`：车道编号加 `1`。
2. 玩家不能越过车道 `0..7`，不能上下移动。每次换道立即将玩家 x 设置为目标车道居中位置。
3. `R` 重开，`ESC` 或关闭窗口退出。

更新顺序
每帧严格执行：处理事件 -> 按固定帧号生成 -> 每个实体 y `+=speed` -> 玩家与实体碰撞 -> 移除顶部大于 `560` 的实体 -> 更新存活分数 -> 绘制。终局后冻结实体和分数。

独特规则
1. 玩家碰到危险块立即失败。
2. 玩家碰到芯片时芯片消失，芯片计数 `+1`，奖励分 `+5`。
3. 存活分固定为 `elapsed_frames // 60`。总分固定为 `存活分 + 芯片数*5`。

绘制顺序与 HUD
1. 绘制背景、游戏区域、`9` 条车道边界线、危险块、芯片、玩家、HUD、终局文字。
2. HUD 字号 `28`：`Lane Cipher` 于 `(20,18)`，`Score: <n>` 于 `(560,18)`，`Chips: <n>` 于 `(560,48)`。
3. 终局字号 `48` 于 `(280,250)` 绘制 `Game Over`，字号 `28` 于 `(285,315)` 绘制 `Press R to Restart`。

禁止项
禁止自由像素移动、上下移动、随机生成、射击、护盾、多生命、道具扩展、关卡和音效。

提交前检查
确认八车道、玩家初始 x、固定生成表、两类实体尺寸、碰撞、计分、HUD、冻结、`R` 和 `ESC` 均已实现。最终只输出代码。
