任务身份
请使用 Python 3 和 pygame 实现单文件游戏 `Prism Relay`。它是网格追踪游戏，不是标准贪吃蛇复刻。必须实现下述信标序列和残留障碍规则，不得套用经典 Snake 默认规则替代。

输出契约
1. 只输出完整 Python 代码，不要输出解释、Markdown 或代码块标记。
2. 禁止读取图片、音频、字体、配置或其他外部文件。只使用 pygame 图元和 `pygame.font.SysFont(None, size)`。
3. 所有可调数值、颜色、固定序列写在文件顶部常量中。调用 `random.seed(42)`，但本任务不得随机生成信标。

坐标与时间
1. 窗口固定为 `800x600`，标题固定为 `Prism Relay`，帧率固定为 `60 FPS`。
2. 左上角为 `(0,0)`。棋盘矩形固定为 `pygame.Rect(100, 90, 600, 480)`。
3. 棋盘固定为 `30` 列 x `24` 行，单格 `20x20`。网格坐标 `(col,row)` 的像素左上角为 `(100 + col*20, 90 + row*20)`。
4. 蛇只在每 `100 ms` 执行一次离散移动。渲染仍保持 `60 FPS`。

颜色表
`BACKGROUND=(11,15,25)`，`BOARD=(20,28,45)`，`GRID=(42,55,78)`，`BORDER=(120,140,170)`，`HEAD=(240,245,255)`，`BODY=(55,210,135)`，`CYAN_BEACON=(40,220,255)`，`AMBER_BEACON=(255,184,64)`，`RESIDUE=(174,92,255)`，`TEXT=(235,240,250)`，`DANGER=(255,84,104)`。

布局与实体
1. 蛇初始网格坐标依次为 `[(15,12),(14,12),(13,12)]`，第一个元素是头，初始方向为右 `(1,0)`。
2. 信标坐标序列固定为 `[(22,12),(22,5),(8,5),(8,18),(25,18),(25,8),(4,8),(4,20),(18,20),(18,3),(27,3),(27,15)]`，循环使用。生成新信标时，从序列当前位置向后找第一个不与蛇身或残留障碍重叠的坐标。
3. 信标颜色严格按吃到次数交替：第 `1,3,5...` 个为 `CYAN_BEACON`，第 `2,4,6...` 个为 `AMBER_BEACON`。
4. 残留障碍候选序列固定为 `[(11,9),(19,14),(6,14),(23,10),(14,18),(16,6)]`。

输入
1. 方向键改变下一次移动方向。
2. 禁止在单次移动中直接反向掉头。
3. `R` 在任意时刻重新初始化全部状态，`ESC` 或关闭窗口立即退出。

更新顺序
每次离散移动严格执行：读取待生效方向 -> 计算新头 -> 检查墙、蛇身、已激活残留障碍 -> 插入新头 -> 处理信标 -> 处理尾部 -> 更新残留倒计时 -> 判断结束。终局后停止离散移动，只处理事件和绘制。

独特规则
1. 吃到任意信标后蛇长度 `+1`。青色信标分数 `+10`，琥珀信标分数 `+20`。
2. 每吃到第 `3` 个信标，创建一个倒计时为 `2` 次离散移动的待激活残留障碍。目标格从残留候选序列中顺序选取第一个空格。
3. 倒计时归零后，在目标格激活 `1x1` 紫色残留障碍。最多保留 `3` 个已激活残留障碍；激活第 `4` 个时删除最早激活的一个。
4. 蛇头碰到墙、蛇身或已激活残留障碍立即失败。不允许穿墙。

绘制顺序与 HUD
1. 依次绘制背景、棋盘、每隔 `20` 像素的网格线、残留障碍、信标、蛇身、蛇头、HUD、终局覆盖文字。
2. HUD 使用字号 `28`，固定绘制 `Prism Relay` 于 `(20,18)`，`Score: <n>` 于 `(560,18)`，`Residues: <n>/3` 于 `(560,50)`。
3. 终局用字号 `48` 在 `(280,255)` 绘制 `Game Over`，字号 `28` 在 `(285,315)` 绘制 `Press R to Restart`。

禁止项
禁止道具、加速、传送、关卡、音效、随机信标、连续像素移动和经典 Snake 的随机食物替代实现。

提交前检查
确认窗口、棋盘、网格尺寸、颜色、固定序列、残留倒计时、三种碰撞终止、HUD、`R`、`ESC` 均已实现。最终只输出代码。
