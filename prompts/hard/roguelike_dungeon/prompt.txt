任务身份
请使用 Python 3 和 pygame 实现单文件游戏 `Rune Vault`。它是双层回合制地牢游戏，但不是随机 Roguelike 模板。地图必须固定，且出口只有在按顺序激活蓝色符文和琥珀符文后才开放。

输出契约
1. 只输出完整 Python 代码，不要输出解释、Markdown 或代码块标记。
2. 禁止外部资源。只使用 pygame 图元和 `pygame.font.SysFont(None, size)`。
3. 所有常量、地图和颜色写在顶部。调用 `random.seed(42)`，但禁止随机生成地图、敌人或道具。

坐标与时间
1. 窗口固定 `800x600`，标题固定 `Rune Vault`，帧率固定 `60 FPS`。
2. 地图左上角固定 `(64,80)`，固定 `21` 列 x `15` 行，单格 `32x32`，像素大小 `672x480`。
3. 网格 `(col,row)` 的像素左上角为 `(64+col*32,80+row*32)`。玩家和敌人占 `1x1` 网格。
4. 游戏采用回合制，只在玩家产生一次有效行动后让全部存活敌人依次行动一次。

颜色表
`BACKGROUND=(10,14,24)`，`WALL=(54,62,82)`，`FLOOR=(24,32,48)`，`PLAYER=(92,218,255)`，`ENEMY=(255,96,116)`，`POTION=(102,232,142)`，`WEAPON=(255,202,84)`，`RUNE_B=(80,210,255)`，`RUNE_A=(255,176,76)`，`EXIT_LOCKED=(126,92,164)`，`EXIT_OPEN=(118,244,156)`，`TEXT=(238,244,255)`。

布局与实体
1. 第一层地图固定为：
```
#####################
#P....#......p......#
#.##..#..#####..##..#
#..b..#......#......#
#.....####...#..e...#
#.........#..#......#
###.#####.#..#####..#
#...#..e..#.........#
#...#.....#####.###.#
#...###........#....#
#p......e..w...#....#
#.#####......###....#
#......a..e.........#
#.................E.#
#####################
```
2. 第二层地图固定为：
```
#####################
#P..e.....#......p..#
#.#####...#..####...#
#.....#...#.........#
#..b..#...#####.##..#
#.....#.........#...#
#.###.#####.###.#...#
#...#...e...#...#...#
#...###.....#...###.#
#p......w...#.......#
#.#####.###.#####...#
#.....#...e.....#...#
#..a..#.......e.....#
#.................E.#
#####################
```
3. `#` 是墙，`.` 是地板，`P` 是玩家出生点，`e` 是四个敌人，`p` 是两个药水，`w` 是一个武器，`b` 是蓝色符文，`a` 是琥珀符文，`E` 是出口。
4. 每进入新层都从对应固定地图重新创建实体和符文状态。
5. 玩家初始属性固定为 `HP=20`、`MAX_HP=20`、`ATK=5`、`LV=1`、`EXP=0`、`Floor=1`。每个敌人初始 `HP=8`、`ATK=3`。

输入
1. 方向键按下尝试向对应网格行动一次。不接受持续按住自动重复。
2. 目标为墙时行动无效，不触发敌人回合。目标为地板、道具、符文或已开启出口时移动有效。目标为敌人时执行近战攻击，也属于有效行动。
3. `R` 重开，`ESC` 或关闭窗口退出。

更新顺序
每次有效行动严格执行：玩家移动或攻击 -> 处理道具 -> 处理符文 -> 处理出口 -> 若未胜利则按敌人列表顺序行动 -> 每只敌人行动后检查玩家 HP -> 检查升级 -> 绘制。终局后冻结。

独特规则
1. 每层出口初始锁定。玩家先踩蓝色符文 `b`，再踩琥珀符文 `a`，出口才永久开放。若在蓝色符文未激活时先踩 `a`，两个符文状态都保持未激活。
2. 药水拾取后消失，恢复 `8` HP，但不超过 `MAX_HP`。武器拾取后消失，`ATK += 2`。
3. 玩家攻击敌人时敌人 `HP -= 玩家ATK`。敌人 HP 小于等于 `0` 时消失，玩家 `EXP += 5`。
4. 每个敌人回合：若与玩家曼哈顿距离为 `1`，玩家 `HP -= 3`；否则尝试向玩家移动一格。候选方向优先级固定 `[上,左,下,右]`，在可通行且未被其他敌人占据的候选中，选择使曼哈顿距离最小的方向；相同距离按优先级。
5. `EXP >= 10` 时立即升级一次：`LV += 1`、`EXP -= 10`、`MAX_HP += 5`、`HP = MAX_HP`、`ATK += 1`。
6. 第一层开放出口后进入第二层；第二层开放出口后胜利。玩家 HP 小于等于 `0` 时失败。

绘制顺序与 HUD
1. 绘制背景、地板、墙、锁定或开放出口、符文、药水、武器、敌人、玩家、HUD、终局文字。
2. 墙填满 `32x32`；地板填满 `32x32`；角色为格内缩 `5` 的 `22x22` 矩形；道具为格中心半径 `8` 圆；符文为格内缩 `8` 的 `16x16` 矩形；出口为格内缩 `4` 的 `24x24` 矩形。
3. HUD 字号 `24`：`Rune Vault` 于 `(20,18)`，`Floor: <n>/2` 于 `(200,18)`，`HP: <n>/<max>` 于 `(330,18)`，`ATK: <n>` 于 `(470,18)`，`LV: <n>` 于 `(570,18)`，`EXP: <n>/10` 于 `(650,18)`。第二行 `(20,48)` 绘制 `Runes: BLUE=<ON|OFF> AMBER=<ON|OFF>`。
4. 终局字号 `48` 于 `(285,250)` 绘制 `You Win` 或 `Game Over`，字号 `28` 于 `(285,315)` 绘制 `Press R to Restart`。

禁止项
禁止随机地图、随机敌人、随机道具、视野迷雾、背包菜单、远程攻击、技能树、多于两层、动画素材和忽略符文顺序的出口。

提交前检查
确认两张地图逐字一致、每层四敌人两药水一武器、有效行动驱动回合、蓝后琥珀、双层出口、战斗、升级、HUD、冻结、`R` 和 `ESC` 均已实现。最终只输出代码。
