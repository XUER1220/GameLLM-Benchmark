请使用 Python 3 和 pygame 实现一个可直接运行的 Tower Defense Hard（塔防）游戏，并严格遵守以下固定规范，以保证不同模型输出的稳定性和可比性。

通用约束：
1. 只输出完整 Python 代码，不要输出解释、Markdown、代码块标记。
2. 代码必须单文件可运行，不能依赖任何本地图片、音频、字体或其他外部资源。
3. 使用 `pygame`，窗口大小固定为 `800x600`，帧率固定为 `60 FPS`。
4. 所有可调参数都写成顶部常量，随机行为使用 `random.seed(42)`。
5. 必须支持关闭窗口退出、`R` 重新开始、`ESC` 退出。
6. 必须显示基地生命、金币、当前波次、剩余敌人、选中塔类型和胜负提示。

玩法与布局固定要求：
1. 主地图固定为 `20 列 x 15 行`，每格 `32x32` 像素，地图区域大小为 `640x480`，位于窗口左侧；右侧 `160` 像素为 HUD 和建造面板。
2. 敌人路径使用固定折线路径，从地图左侧入口到右侧基地，不要随机生成。路径必须覆盖多个转弯点，并在地图上清晰绘制。
3. 玩家使用鼠标在非路径格子上建塔；使用 `1/2/3` 选择塔类型，左键建造，右键升级已有塔。不能在路径、基地、入口或已有塔位置建塔。
4. 固定实现 `3` 种塔：
   - Arrow Tower：花费 `50`，单体攻击，射程 `120`，伤害 `8`，射速 `0.8` 秒
   - Cannon Tower：花费 `80`，范围溅射，射程 `105`，伤害 `14`，射速 `1.2` 秒，溅射半径 `45`
   - Slow Tower：花费 `70`，低伤害减速，射程 `110`，伤害 `4`，射速 `1.0` 秒，命中后减速 `2` 秒
5. 升级塔固定花费为原始价格的 `70%`，每座塔最多升级 `2` 次；每次升级提升伤害和射程，并在 HUD 或塔身上显示等级。
6. 敌人沿路径连续移动，不能穿越路径外区域。敌人到达基地时基地生命 `-1`，敌人消失。
7. 固定实现 `5` 波敌人，每波数量和强度逐步提升。至少包含普通、快速、重甲三类敌人，分别体现不同速度、生命值和奖励金币。
8. 初始金币固定为 `180`，基地生命固定为 `20`。击败敌人获得金币奖励。
9. 每波之间有 `3` 秒准备时间；当前波全部生成并被击败后进入下一波。
10. 所有 `5` 波敌人被击败且基地生命大于 `0` 时判定胜利；基地生命归零时判定失败。
11. 游戏结束后显示 `You Win` 或 `Game Over`、最终波次和 `Press R to Restart`。
12. 不要加入随机地图、拖拽路径编辑、技能卡牌、联网、音效或外部素材。

视觉要求：
1. 地面、路径、入口、基地、三种塔、三类敌人、炮弹/箭矢、HUD 颜色要明显区分。
2. 塔的射程在鼠标悬停或选中时可视化，攻击弹道或命中特效应能体现塔正在工作。
