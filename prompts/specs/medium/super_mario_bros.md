## 任务身份
请使用 Python 3 和 pygame 实现单文件游戏 `Beacon Courier`。它是固定横向卷轴平台游戏，不是 Super Mario Bros 关卡复刻。出口旗杆只有在收集四个电池后才会激活。

## 坐标与时间
1. 窗口固定 `800x600`，标题固定 `Beacon Courier`，帧率固定 `60 FPS`。
2. 世界固定宽 `3200`、高 `600`。屏幕坐标 x 固定为 `world_x-camera_x`。摄像机固定为 `clamp(player_center_x-400,0,2400)`。
3. 所有碰撞使用世界坐标整数 `pygame.Rect`。玩家竖直速度允许浮点数，绘制时使用 `round()`。

## 颜色表
`SKY=(18,38,66)`，`GROUND=(92,72,58)`，`PLATFORM=(164,112,72)`，`QUESTION=(255,188,72)`，`PIPE=(72,188,112)`，`PLAYER=(92,190,255)`，`ENEMY=(255,104,96)`，`COIN=(255,220,76)`，`BATTERY=(176,104,255)`，`FLAG_LOCKED=(124,136,158)`，`FLAG_ACTIVE=(92,236,142)`，`TEXT=(242,246,255)`。

## 布局与实体
1. 玩家矩形固定 `32x48`，初始世界矩形 `pygame.Rect(80,472,32,48)`。水平速度固定 `5` 像素/帧，跳跃初速度固定 `-12`，重力固定每帧 `+0.5`，最大下落速度 `12`。
2. 地面矩形固定为 `[(0,520,880,80),(1000,520,660,80),(1780,520,740,80),(2640,520,560,80)]`。四段之间形成三个坑洞。
3. 空中平台矩形固定为 `[(360,420,40,32),(400,420,40,32),(440,420,40,32),(760,360,40,32),(800,360,40,32),(1160,420,40,32),(1200,420,40,32),(1240,420,40,32),(1880,390,40,32),(1920,390,40,32),(1960,390,40,32),(2360,350,40,32)]`。
4. 管道矩形固定为 `[(620,456,64,64),(1400,424,64,96),(2240,456,64,64)]`。
5. 结尾台阶矩形固定为 `[(2740,488,40,32),(2780,456,40,64),(2820,424,40,96),(2860,392,40,128)]`。
6. 金币固定为 `18x18`，左上角坐标固定为 `[(371,390),(411,390),(771,330),(811,330),(1171,390),(1211,390),(1510,470),(1891,360),(1931,360),(2070,470),(2371,320),(2690,470)]`。
7. 电池固定为 `24x24`，左上角坐标固定为 `[(505,470),(805,326),(1948,356),(2470,470)]`。
8. 三个敌人固定 `32x32`，初始左上角与巡逻闭区间固定为 `(740,488,[700,840])`、`(1510,488,[1470,1600])`、`(2130,488,[2040,2200])`，初始水平速度均为 `-2`。
9. 旗杆固定矩形 `pygame.Rect(3060,320,12,200)`。

## 输入
1. 持续按左/右方向键水平移动。玩家在地面或平台上时按下 `SPACE` 才能跳跃。
2. `R` 重开，`ESC` 或关闭窗口退出。

## 更新顺序
每帧严格执行：处理事件 -> 水平移动并逐轴解决固体碰撞 -> 重力和竖直移动并逐轴解决碰撞 -> 更新敌人巡逻 -> 处理金币 -> 处理电池 -> 处理敌人踩踏或侧碰 -> 处理跌落 -> 更新安全重生点 -> 更新摄像机 -> 检查旗杆 -> 绘制。终局后冻结。

## 独特规则
1. 出口旗杆只有在四个电池全部收集后才由灰色变绿色；未激活旗杆是不可穿越的固体矩形。
2. 安全重生点初始为 `(80,472)`；玩家世界 x 首次大于等于 `1080` 后改为 `(1040,472)`；首次大于等于 `1840` 后改为 `(1820,472)`；首次大于等于 `2680` 后改为 `(2660,472)`。
3. 失去生命后玩家回最近安全重生点，速度归零。
4. 玩家从上方下落且上一帧底部小于等于敌人顶部时碰敌人，才按踩踏处理；其他敌人碰撞按受伤处理。

## 计分与终止
1. 金币接触后消失，金币数 `+1`，分数 `+100`。电池接触后消失，电池数 `+1`，分数 `+250`。
2. 从上方踩踏敌人时敌人消失，分数 `+200`，玩家竖直速度设为 `-7`；其他敌人碰撞生命 `-1`。
3. 玩家跌落到 `y>600` 时生命 `-1`。生命总数固定 `3`，生命为 `0` 时失败。
4. 只有电池数达到 `4` 后接触旗杆才胜利，否则旗杆保持锁定。

## 绘制顺序与 HUD
1. 绘制天空、地面、平台、管道、台阶、金币、电池、敌人、旗杆、玩家、固定屏幕 HUD、终局文字。
2. HUD 字号 `26`：`Beacon Courier` 于 `(20,18)`，`Lives: <n>` 于 `(250,18)`，`Score: <n>` 于 `(380,18)`，`Coins: <n>` 于 `(540,18)`，`Batteries: <n>/4` 于 `(650,18)`。
3. 终局字号 `48` 于 `(285,250)` 绘制 `You Win` 或 `Game Over`，字号 `28` 于 `(285,315)` 绘制 `Press R to Restart`。

## 禁止项
禁止随机关卡、原版地图、原版素材、变大、火球、隐藏砖、多关卡、音效、复杂动画和未收集电池即可通关的实现。
