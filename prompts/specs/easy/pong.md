## 任务身份
请使用 Python 3 和 pygame 实现单文件游戏 `Gate Rally`。它保留双人球拍对打，但不是标准 Pong 复刻：出界球必须命中当前高亮得分门才计分。

## 坐标与时间
1. 窗口固定 `800x600`，标题固定 `Gate Rally`，帧率固定 `60 FPS`。
2. 对打区域固定为 `pygame.Rect(40,80,720,480)`。所有实体使用整数 `pygame.Rect`。

## 颜色表
`BACKGROUND=(10,16,28)`，`FIELD=(20,30,48)`，`LINE=(88,108,138)`，`LEFT_PADDLE=(80,220,255)`，`RIGHT_PADDLE=(255,166,82)`，`BALL=(245,248,255)`，`ACTIVE_GATE=(132,255,152)`，`INACTIVE_GATE=(62,78,102)`，`TEXT=(238,244,255)`，`WIN=(255,224,96)`。

## 布局与实体
1. 左球拍固定初始矩形 `pygame.Rect(64,272,16,96)`，右球拍固定初始矩形 `pygame.Rect(720,272,16,96)`。
2. 球固定大小 `16x16`，每回合初始矩形 `pygame.Rect(392,312,16,16)`。
3. 左右后墙各有三个门，矩形分别为 `(40,112,8,96)`、`(40,272,8,96)`、`(40,432,8,96)` 和 `(752,112,8,96)`、`(752,272,8,96)`、`(752,432,8,96)`。
4. 当前高亮门索引固定为 `serve_count % 3`。每次球出界后 `serve_count += 1`。
5. 发球水平符号序列固定循环 `[1,-1,-1,1]`，竖直符号序列固定循环 `[1,-1,1,-1]`。球初速度固定为 `(5*水平符号,4*竖直符号)`。

## 输入
1. 左球拍持续按住 `W/S` 上下移动，右球拍持续按住上/下方向键上下移动。
2. 球拍速度固定 `7` 像素/帧，球拍必须限制在区域 `y=80..464`。
3. `R` 重开，`ESC` 或关闭窗口退出。

## 更新顺序
每帧严格执行：处理事件 -> 更新球拍 -> 更新球位置 -> 处理上下边界反弹 -> 处理球拍碰撞 -> 处理左右出界和得分门 -> 检查胜利 -> 绘制。终局后冻结。

## 独特规则
1. 左右后墙各有三个得分门，当前高亮门按目标门序列轮换。
2. 出界球只有命中当前高亮门才计分；非高亮位置出界不得分但仍重新发球。
3. 球与球拍碰撞时只反转 x 速度，并把球夹到球拍外侧，不加入旋转、加速或随机偏移。

## 计分与终止
1. 球越过左边界 `x<40` 时，只有球中心 y 落在左侧当前高亮门的 `[top,bottom)` 内，右方才 `+1` 分。
2. 球越过右边界 `x>760` 时，只有球中心 y 落在右侧当前高亮门的 `[top,bottom)` 内，左方才 `+1` 分。
3. 球从非高亮位置出界时双方均不得分，但仍开始下一次发球。
4. 任一方先到 `7` 分获胜，终局后冻结。

## 绘制顺序与 HUD
1. 绘制背景、对打区域、中线、六个门、高亮门、球拍、球、HUD、终局文字。
2. HUD 字号 `34`：左分于 `(300,24)`，右分于 `(480,24)`；字号 `24`：`Gate Rally` 于 `(20,22)`，`Target Gate: <1|2|3>` 于 `(600,24)`。
3. 终局字号 `44` 于 `(275,250)` 绘制 `Left Wins` 或 `Right Wins`，字号 `28` 于 `(285,315)` 绘制 `Press R to Restart`。

## 禁止项
禁止 AI 球拍、音效、菜单、旋转物理、随机发球、多球和忽略得分门的经典计分逻辑。

## 提交前检查
确认六个门、轮换高亮门、非高亮出界不计分、固定发球序列、边界、HUD、冻结、`R` 和 `ESC` 均已实现。
