## 任务身份
请使用 Python 3 和 pygame 实现单文件游戏 `Pulse Gates`。它是单键飞行穿门游戏，不是标准 Flappy Bird 复刻。玩家必须触碰每个门内的脉冲球，不能仅穿过空隙。

## 坐标与时间
1. 窗口固定 `800x600`，标题固定 `Pulse Gates`，帧率固定 `60 FPS`，左上角为 `(0,0)`。
2. 飞行区域为 `pygame.Rect(0,70,800,490)`。地面装饰条为 `pygame.Rect(0,560,800,40)`，属于碰撞边界。
3. 内部速度允许浮点数；绘制和 `pygame.Rect` 碰撞时使用 `round()` 转为整数。

## 颜色表
`SKY=(14,30,54)`，`GROUND=(58,72,82)`，`BIRD=(255,214,72)`，`BIRD_WING=(255,142,72)`，`PIPE=(72,194,116)`，`PIPE_EDGE=(32,126,82)`，`ORB=(80,226,255)`，`ORB_RING=(235,250,255)`，`TEXT=(242,246,255)`，`DANGER=(255,86,108)`。

## 布局与实体
1. 鸟碰撞盒固定为 `pygame.Rect(150,284,42,30)`。
2. 每组门由上下两个管道和一个脉冲球组成。管道宽固定 `72`，间隙高固定 `168`。
3. 门的间隙中心序列固定为 `[190,330,240,380,270,160,350,220]`，循环使用。
4. 第一组门在第 `45` 帧生成，之后每隔 `100` 帧生成一组，生成时管道左边固定为 `x=800`。
5. 对于中心 `center_y`，上管道矩形固定为 `(x,70,72,center_y-84-70)`，下管道矩形固定为 `(x,center_y+84,72,560-(center_y+84))`。
6. 脉冲球矩形固定为 `(x+24,center_y-12,24,24)`。每组门独立记录 `orb_collected`。

## 输入
1. `SPACE` 或上方向键按下时，将鸟的竖直速度立即设为 `-7.2`。
2. `R` 重开，`ESC` 或关闭窗口退出。

## 更新顺序
每帧严格执行：处理事件 -> 竖直速度 `+=0.36` -> 鸟 y `+=速度` -> 生成门 -> 所有门 x `-=3` -> 检测鸟与脉冲球 -> 检测鸟与管道或上下边界 -> 检测漏球 -> 移除右边小于 `0` 的门 -> 绘制。终局后冻结实体。

## 独特规则
1. 每组门内的脉冲球是强制目标；只穿过管道空隙不能视为通过。
2. 每组门维护 `orb_collected` 状态，鸟第一次触碰该组脉冲球时球消失并标记已收集。
3. 若门越过鸟仍未收集脉冲球，失败原因必须显示为 `Missed Orb`。

## 计分与终止
1. 鸟第一次碰到某组门的脉冲球时，该球消失，分数 `+1`。
2. 若某组门的右边严格小于鸟的左边，且该组 `orb_collected=False`，立即失败并显示 `Missed Orb`。
3. 鸟与任一管道矩形碰撞，或鸟顶部小于 `70`，或鸟底部大于 `560`，立即失败并显示 `Game Over`。
4. 终局后冻结实体和分数，只保留事件处理、绘制、`R` 和 `ESC`。

## 绘制顺序与 HUD
1. 依次绘制天空、管道、管道边框、未收集脉冲球、鸟、地面、HUD、终局文字。
2. HUD 字号 `30`：`Pulse Gates` 于 `(20,18)`，`Score: <n>` 于 `(650,18)`。
3. 终局字号 `48` 于 `(270,250)` 绘制失败原因，字号 `28` 于 `(285,315)` 绘制 `Press R to Restart`。

## 禁止项
禁止随机门高、图片、音效、动画素材、道具、敌人、关卡、菜单和仅按穿过管道计分的经典规则。

## 提交前检查
确认脉冲球是强制目标，漏球会失败，所有尺寸、坐标、颜色、速度、生成帧、HUD、终止和重开逻辑均与规格一致。
