请使用 Python 3 和 pygame 实现一个可直接运行的 Pac-Man Medium（吃豆人）游戏，并严格遵守以下固定规范，以保证不同模型输出的稳定性和可比性。

通用约束：
1. 只输出完整 Python 代码，不要输出解释、Markdown、代码块标记。
2. 代码必须单文件可运行，不能依赖任何本地图片、音频、字体或其他外部资源。
3. 使用 `pygame`，窗口大小固定为 `800x600`，帧率固定为 `60 FPS`。
4. 所有可调参数都写成顶部常量，随机行为使用 `random.seed(42)`。
5. 必须支持关闭窗口退出、`R` 重新开始、`ESC` 退出。
6. 必须显示分数、剩余生命、剩余豆子数量和胜负提示。

玩法与布局固定要求：
1. 迷宫区域固定为 `19 列 x 21 行`，每格 `24x24` 像素，迷宫显示区域大小为 `456x504`，在 `800x600` 窗口中居中偏左显示，右侧保留 HUD 区域。
2. 使用固定二维字符串或列表定义迷宫，不要随机生成。必须包含墙体、通路、普通豆子、能量豆、玩家出生点、幽灵出生区和左右贯通隧道。
3. 玩家占 `1` 个网格，使用方向键进行四方向移动，移动速度固定为每帧 `3` 像素；玩家只能在通路中移动，不能穿墙。
4. 普通豆子被吃掉后消失并得分 `+10`；能量豆固定为 `4` 个，被吃掉后得分 `+50`，并让所有幽灵进入可被吃状态 `6` 秒。
5. 幽灵固定为 `4` 个，颜色明显区分。幽灵从出生区出发，使用确定性追逐/巡逻逻辑：优先朝玩家方向移动，遇到岔路时可根据固定顺序选择方向，但必须避免穿墙。
6. 玩家在普通状态碰到幽灵时失去 `1` 条生命，并重置玩家与幽灵到出生位置；玩家总生命固定为 `3`。
7. 能量豆生效期间，玩家碰到幽灵应吃掉幽灵并得分 `+200`，被吃幽灵回到出生区，能量状态结束后恢复普通危险状态。
8. 所有豆子被吃完时判定胜利；生命归零时判定失败。
9. 游戏结束后显示 `You Win` 或 `Game Over`、最终分数和 `Press R to Restart`。
10. 不要加入水果奖励、多个关卡、复杂幽灵人格、音效或动画帧素材。

视觉要求：
1. 墙体、通路、普通豆子、能量豆、玩家、幽灵、HUD 颜色要明显区分。
2. 迷宫边界完整清晰，豆子与角色不能被墙体遮挡。
