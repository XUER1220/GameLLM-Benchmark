任务身份
请使用 Python 3 和 pygame 实现单文件游戏 `Switch Maze`。它是固定迷宫追逐游戏，但不是标准 Pac-Man 复刻。迷宫包含必须依次触发的蓝色和琥珀色开关门。

输出契约
1. 只输出完整 Python 代码，不要输出解释、Markdown 或代码块标记。
2. 禁止外部资源。只使用 pygame 图元和 `pygame.font.SysFont(None, size)`。
3. 所有常量、地图和颜色写在顶部。调用 `random.seed(42)`，但地图和幽灵决策不得随机。

坐标与时间
1. 窗口固定 `800x600`，标题固定 `Switch Maze`，帧率固定 `60 FPS`。
2. 迷宫左上角固定 `(40,72)`，固定 `19` 列 x `21` 行，单格 `24x24`，像素大小 `456x504`。
3. 网格 `(col,row)` 的像素左上角固定为 `(40+col*24,72+row*24)`。角色使用网格离散移动，每次有效移动间隔固定 `120 ms`。

颜色表
`BACKGROUND=(8,13,26)`，`WALL=(48,92,220)`，`PATH=(12,20,38)`，`PELLET=(245,232,184)`，`POWER=(255,184,72)`，`PLAYER=(255,226,74)`，`GHOSTS=[(255,92,112),(82,222,255),(255,154,222),(255,168,74)]`，`SWITCH_A=(70,210,255)`，`SWITCH_B=(255,182,72)`，`DOOR_CLOSED=(182,104,255)`，`DOOR_OPEN=(74,98,128)`，`TEXT=(238,244,255)`。

布局与实体
1. 地图必须逐行使用以下 `21` 个等长字符串，不得修改：
```
###################

#.##.#.###.#.#.##.#
#....#.....#.#....#
####.###A###.###.##
#.......#.#.......#
#.###.#.#.#.#.###.#

_____.#..P..#._____
#####.#.....#.#####
#.....###B###.....#
#.###.....a.....#.#
#.#.#####.#####.#.#
#.#.....#.#.....#.#
#.#####.#.#.#####.#
#.......#b#.......#
#.##.###.#.###.##.#
#o...#.......#...o#
#.................#
###################
```
2. `#` 是墙，`.` 是普通豆，`o` 是能量豆，`P` 是玩家出生点，`G` 是四个幽灵出生点，`a/b` 是开关，`A/B` 是门，`_` 是左右隧道外部通路。
3. 玩家和幽灵占 `1x1` 网格。幽灵移动间隔固定 `180 ms`。

输入
1. 方向键设置玩家期望方向。到达下一个移动时刻后，仅在目标格可通行时移动；否则保持原位。
2. `R` 重开，`ESC` 或关闭窗口退出。

更新顺序
每次玩家移动时刻：应用期望方向 -> 处理豆和开关 -> 处理隧道横向传送 -> 检查幽灵碰撞。每次幽灵移动时刻：按幽灵编号顺序移动 -> 每只移动后检查碰撞。每帧再更新能量豆倒计时、检查胜负并绘制。终局后冻结。

独特规则
1. 门 `A` 和 `B` 初始关闭，关闭时视为墙。玩家踩到开关 `a` 后永久打开门 `A`。
2. 玩家只有在门 `A` 已打开后踩到开关 `b`，才永久打开门 `B`。若先踩到 `b`，不得开门。
3. 普通豆消失并 `+10` 分。四个能量豆消失并 `+50` 分，使幽灵可被吃 `6000 ms`。
4. 四只幽灵的岔路优先级分别固定为 `[上,左,下,右]`、`[左,下,右,上]`、`[下,右,上,左]`、`[右,上,左,下]`。优先选择使曼哈顿距离更接近玩家的可通行方向；相同距离按各自优先级。幽灵不能穿墙或关闭的门。
5. 普通状态碰幽灵：生命 `-1`，玩家和幽灵回出生点。总生命固定 `3`。能量状态碰幽灵：幽灵回出生点，分数 `+200`。
6. 所有普通豆和能量豆清空则胜利；生命为 `0` 则失败。

绘制顺序与 HUD
1. 绘制背景、通路、墙、门、豆、开关、幽灵、玩家、右侧 HUD、终局文字。
2. 墙填满 `24x24`；普通豆为格中心半径 `3` 圆；能量豆半径 `7`；角色为格中心半径 `9` 圆；门为格内缩 `2` 的 `20x20` 矩形。
3. HUD 字号 `26` 固定在右侧：`Switch Maze` 于 `(530,90)`，`Score: <n>` 于 `(530,140)`，`Lives: <n>` 于 `(530,180)`，`Pellets: <n>` 于 `(530,220)`，`Door A: OPEN|LOCKED` 于 `(530,280)`，`Door B: OPEN|LOCKED` 于 `(530,320)`。
4. 终局字号 `48` 于 `(520,400)` 绘制 `You Win` 或 `Game Over`，字号 `24` 于 `(520,460)` 绘制 `Press R to Restart`。

禁止项
禁止随机迷宫、随机幽灵、水果、多关卡、素材、音效、动画帧和忽略顺序开门的经典迷宫替代实现。

提交前检查
确认地图逐字一致、尺寸换算、四个能量豆、四只幽灵、门 A 后门 B、隧道、碰撞、HUD、冻结、`R` 和 `ESC` 均已实现。最终只输出代码。
